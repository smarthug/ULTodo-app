## 1. 비전

> **"덜 고른다"가 의지력으로 싸워야 하는 미덕이 아니라 기본 동작이 되도록 만드는 생산성 앱.**

대부분의 사람은 task를 *담아둘 곳*이 부족한 게 아닙니다. 하루의 시작에서 정말 중요한 소수의 일을 떠올리고, 집중해서 실행하도록 강제하는 *짧고 의견 있는 데일리 루프*가 부족할 뿐입니다. ULTodo는 그 루프를 제품 안에 박아 넣었습니다.

> *프릭션 없이 캡처 → 우선도로 분류 → 작은 Today 리스트 선택 → 조용한 타이머 단위로 실행*

이 앱은 자기 자신의 디바이스에서 혼자 쓰는 사람을 위해 설계되었습니다. 팀 모드 없음, 동기화 없음, 분석 없음, growth surface 없음. SaaS가 아니라 *생각하기 위한 도구*입니다.

## 2. 문제 정의

### 2.1 사용자의 통증
지식 노동자와 자기 주도형 개인은 노트, 포스트잇, 프로젝트 보드, 이메일 인박스에 걸쳐 수십 개의 "열린" task를 일상적으로 쌓아둡니다. 결과는 만성적인 *낮은 강도의 분류 부채(triage debt)* 상태입니다. 매일 아침은 "일하기"가 아니라 "무엇을 할지 결정하기"로 시작됩니다.

### 2.2 기존 도구가 부족한 이유
- **범용 task 매니저** (Todoist, Things, Apple Reminders): 캡처는 잘하지만 그날의 집중 의식이 약합니다. "오늘" 리스트는 "오늘 마감인 것"인데, 그게 우선도와 일치하는 경우는 드뭅니다.
- **프로젝트 트래커** (Notion, Linear, Asana): 협업과 구조의 오버헤드가 단일 사용자 생산성에는 불필요합니다.
- **포모도로 앱** (Forest, Be Focused): 실행은 다루지만 task 리스트와 분리되어 있어, 컨텍스트 스위칭이 발생합니다.
- **아이젠하워 매트릭스 도구**: 보통 정적인 템플릿이지, 인터랙티브한 분류 표면이 아닙니다.

이 다섯 가지 — 프릭션 없는 캡처 + 매트릭스 분류 + 작은 강제 함수 "today" 리스트 + 통합된 타이머 실행 + 오프라인 우선 단일 사용자 모델 — 를 한 곳에 결합한 도구는 없습니다.

### 2.3 기회
이 워크플로우들을 사용자의 *주의력 예산*을 존중하는 방식으로 결합. 결정 피로를 없애기 위해 의견 있는 기본값 사용. 계정/동기화 마찰을 없애기 위해 오프라인 전용. 이력서용 기능 체크가 아니라 *매일 다시 쓰는 것*에 최적화.

## 3. 타깃 페르소나

### 페르소나 A — 독립 메이커
- 솔로 창업가, 프리랜서, 인디 크리에이터, 또는 대학원생
- 자기 시간을 통제하지만 우선순위를 정해줄 상사가 없음
- 4개 이상의 task 매니저를 시도했고 한 달 내에 모두 포기함
- 대부분의 일에 "아니오"라고 말하고 *지금 한 가지에* "예"라고 말하도록 도와주는 도구를 원함
- **멘탈 모델**: "기능이 더 필요한 게 아니에요. 다음 일을 *시작하는 게* 필요한 거지."

### 페르소나 B — 집중하는 지식 노동자
- 큰 조직 안의 IC 엔지니어, 디자이너, 작가, 또는 분석가
- 팀 작업은 이미 Linear/Jira에 산다; 개인 집중을 위한 *분리된* 개인 레이어가 필요
- 개인 todo는 절대 동기화되지 않고, 디바이스를 떠나지 않으며, 어떤 피드에도 나타나지 않기를 원함
- **멘탈 모델**: "팀 트래커는 팀을 위한 거야. 이건 나를 위한 거야."

### 페르소나 C — 프라이버시 의식 사용자 (보조)
- task 리스트가 클라우드에서 완전히 떨어져 있길 원함
- 오프라인 우선, 무계정, 무텔레메트리를 *하드 요건*으로 봄
- 그것을 위해 더 작은 기능 표면을 받아들임
- **멘탈 모델**: "서버에 1바이트라도 보내면 내 도구가 아니야."

## 4. 제품 원칙

1. **모바일 우선, 데스크탑은 부가.** 430px 폰 프레임 레이아웃이 정본 경험; 데스크탑은 그것을 펼친 모양일 뿐, 별개의 제품이 아님.
2. **기본은 오프라인 전용.** 계정 없음, 동기화 없음, 텔레메트리 없음. 디바이스의 IndexedDB가 유일한 진실의 출처.
3. **한 번에 한 화면.** 중첩 모달 없음, 탭 안의 탭 없음. 모바일에서는 시트가 위로 슬라이드, 데스크탑에서는 인라인 패널.
4. **의견 있는 기본값, 부드러운 탈출구.** Today는 기본 3개 task. 포모도로는 기본 25/5분. 미설정 우선도의 암묵적 폴백은 Inbox. Settings는 존재하지만 대부분의 사용자는 거기 안 감.
5. **조용한 타이포그래피.** 시각 언어는 *대시보드*가 아니라 *종이*. 헤드라인은 Instrument Serif 이탤릭, 본문은 Geist, 숫자는 Geist Mono.
6. **캡처에서의 마찰은 적.** 새 task는 Apple Notes 여는 것보다 적은 키 입력으로 만들어져야 함. Quick add는 어느 화면에서든 한 번의 탭.
7. **노력 없는 온보딩.** 첫 로드 시 매트릭스와 브레인덤프가 채워져 보이도록 충분한 프로젝트, 태그, 예제 task가 시드됨. 사용자는 지우거나 유지할 수 있음.

## 5. 사용자 스토리

### 5.1 데일리 루프 (P0 — 필수)
- **US-01** 사용자로서, 앱을 열면 진행률 바와 함께 Today 짧은 리스트가 보여서 오늘 정확히 무엇을 실행할지 안다.
- **US-02** 사용자로서, Today task의 `Start`를 탭하면 카운트다운 아래에 그 task의 제목이 표시된 채 포모도로 타이머가 띄워진다.
- **US-03** 사용자로서, 어느 화면에서든 5초 안에 새 task를 캡처한다 — 모바일은 바텀바 `+`, 데스크탑은 사이드바 Add task 버튼.
- **US-04** 사용자로서, 캡처한 task를 Inbox에서 4개 매트릭스 사분면 중 하나로 드래그해서 우선도를 부여한다.
- **US-05** 사용자로서, 어느 리스트 view에서든 체크박스 한 번 탭으로 task를 완료 표시한다.

### 5.2 분류 및 리뷰 (P0)
- **US-06** 사용자로서, Brain Dump를 열면 그룹된(Inbox / Important / Someday / Completed) 모든 task가 보여서 한 번에 훑고 분류할 수 있다.
- **US-07** 사용자로서, Brain Dump를 카드 view로 전환하면 데스크탑에서 3컬럼 카드로 시각적으로 훑을 수 있다.
- **US-08** 사용자로서, 제목/노트 텍스트로 Brain Dump를 검색해서 특정 task를 찾는다.
- **US-09** 사용자로서, Brain Dump와 Today를 프로젝트 그리고/또는 태그로 필터링해서 view 범위를 좁힌다.

### 5.3 설정 (P1 — 권장)
- **US-10** 사용자로서, Today 집중 개수를 3개에서 N개(1~7)로 바꿔서 데일리 루프가 실제 캐파시티에 맞게 한다.
- **US-11** 사용자로서, 포모도로 집중·휴식 분을 선호하는 타이밍(예: 50/10)으로 바꾼다.
- **US-12** 사용자로서, Settings에서 UI 전체를 영어와 한국어 사이로 전환한다.
- **US-13** 사용자로서, Settings → Projects에서 이름과 색을 가진 새 사용자 정의 프로젝트를 만든다.
- **US-14** 사용자로서, 기존 프로젝트 이름을 바꾸거나 보관(archive)한다. 기본 `Personal` 프로젝트는 보관할 수 없다.

### 5.4 데스크탑 인체공학 (P1)
- **US-15** 데스크탑 사용자로서, Today 리스트와 task detail 패널을 나란히 보아서, 편집할 때 리스트에서 벗어나지 않는다.
- **US-16** 데스크탑 사용자로서, 모든 4개 매트릭스 사분면을 2×2가 아니라 한 줄로 본다.
- **US-17** 데스크탑 사용자로서, Today / Brain / Matrix / Pomo / Settings 사이를 바텀 탭이 아니라 영구적인 좌측 사이드바로 이동한다.

### 5.5 편집 및 다듬기 (P1)
- **US-18** 사용자로서, 어느 view에서든 task를 탭하면 focus, 완료 표시, 포모도로 시작, 사분면 변경, 전체 편집 폼 열기 옵션과 함께 그 task의 전체 detail이 보인다.
- **US-19** 사용자로서, 한 인라인 폼에서 task의 제목, 노트, 프로젝트, 태그, 우선도, focus 플래그를 편집한다.
- **US-20** 사용자로서, 현재 선택된 사분면 버튼을 클릭해서 task를 Inbox로 되돌린다 (토글 동작).

## 6. 기능 요건

### 6.1 라우팅 & 셸
- 5개 라우트: `/today`, `/brain`, `/matrix`, `/pomo`, `/settings`
- 기본 라우트는 `/today`
- `AppShell`이 AppBar, BottomNav (모바일), Sidebar (데스크탑)을 제공하고 활성 페이지를 `<Outlet />`에 렌더
- 단일 `useIsDesktop()` 훅(`(min-width: 1024px)`)이 반응형 분기를 결정

### 6.2 Today 페이지
- 짧은 리스트로 `selectTodayTasks(tasks, settings)`를 읽음 (결정적, `settings.todayCount`로 캡됨)
- 완료/전체를 보여주는 라이브 진행률 바
- 각 task는 `Start` 버튼 오버레이가 있는 `TaskCard`
- 페이지 헤더 안에 인라인 focus count 스테퍼 (`Minus` / 값 / `Plus`)
- 데스크탑: 3-pane 레이아웃 (필터 칩 헤더 + 리스트 / 우측 `TaskDetailPanel`)

### 6.3 Brain Dump 페이지
- 두 view 모드: `list`(그룹 섹션)과 `card`(모바일 1컬럼, 데스크탑 3컬럼 그리드)
- 검색 input은 제목과 노트로 필터링 (대소문자 무시)
- View 모드는 `settings.brainView`에 영속됨
- 데스크탑: Today와 같은 3-pane 모양 (필터 칩 헤더 + 리스트/그리드 + 우측 `TaskDetailPanel`)

### 6.4 Eisenhower Matrix 페이지
- 2×2 사분면 그리드 (모바일) 또는 4컬럼 한 줄 (데스크탑, ≥1024px)
- 그리드 아래에 풀폭 Inbox 스트립
- `@dnd-kit/core`를 통한 드래그앤드롭: `PointerSensor`(5px 활성화 거리) + `TouchSensor`(120ms 지연, 8px 허용오차)
- `DragOverlay` 칩 너비: 모바일 192px, 데스크탑 144px
- 사분면은 좁은 너비에서 텍스트가 truncate 되는 `MatrixTaskChip` 항목을 렌더
- 폴백: 칩을 탭하면 → `TaskDetailPanel` 열림 → "Move without dragging" 4-버튼 토글 섹션

### 6.5 Pomodoro 페이지
- 두 모드: focus, break (토글 pill)
- 시간 길이는 `settings.pomodoroMinutes`와 `settings.breakMinutes`에서 가져옴
- 원형 SVG 모방 진행률 (테두리가 있는 링의 clip-path)
- 모바일: 288px 원. 데스크탑: 384px 원.
- 라우트가 `navigate('/pomo', { state: { taskId } })`로 도달했다면 (`Start` 버튼에서) 그 task를 focus 주제로 사전 로드
- 그렇지 않으면 `focus: true`이고 `done: false`인 첫 task를 픽
- 컨트롤: Reset / Start / Pause

### 6.6 Settings 페이지
- 언어 토글: en / ko (`settings.language`에 쓰고 `i18n.changeLanguage` 호출)
- Today 집중 개수 스테퍼: 1~7
- 포모도로 스테퍼: 집중 5~60, 휴식 1~30
- **프로젝트 관리**: 이름 변경/보관 버튼이 있는 비보관 프로젝트 리스트 + 인라인 신규 프로젝트 폼 (이름 + 8색 팔레트 swatch picker)
- Personal 프로젝트의 보관 버튼은 숨김 (보관 불가)
- 하단에 scope hint 단락

### 6.7 Quick add / 편집 폼 (`QuickAddForm`)
- 신규 task 생성과 기존 task 편집 모두에 쓰이는 단일 공유 컴포넌트
- 자체 상태 보유 (8개 useState 필드). 부모는 `open`-keyed `useEffect`가 아니라 React `key` prop으로 reset 제어
- 필드: 제목, 노트, 프로젝트(드롭다운), 추정 시간(숫자), 태그(칩 + 새 태그 input), 우선도(4개 토글 사분면 버튼, Inbox 버튼 없음 — 미선택 = Inbox), Today focus 토글
- 모바일 진입: 바텀 시트 (`QuickAddSheet`)
- 데스크탑 진입: Sidebar `Add task` 버튼에서 트리거 시 Today/BrainDump 리스트 상단에 인라인 패널

### 6.8 Task detail 패널 (`TaskDetailPanel`)
- 두 variant: `sheet`(모바일 바텀 시트)와 `inline`(데스크탑 우측 고정 패널)
- 인라인 variant는 task가 선택되지 않았을 때 "Select a task" 빈 상태 표시
- 본문: 제목, 노트, Focus / Done / Start 버튼(3컬럼), Move without dragging(4버튼 토글, Inbox 버튼 없음), Edit task 버튼
- Edit task → `QuickAddForm` 열림 (모바일은 시트, 데스크탑은 본문 아래 인라인)
- 방어적 lookup: `store.tasks.find(t => t.id === selectedTask?.id) ?? null` — 삭제 시 패널 자동 클리어

### 6.9 영속성
- 단일 IndexedDB 데이터베이스 `ultodo`, 오브젝트 스토어: `tasks`, `projects`, `tags`, `settings`, `meta`
- `bootstrapDatabase()`가 첫 로드 시 기본값을 시드 (`meta.seededVersion`을 통해 멱등)
- 모든 변경은 `repositories.ts` export → `useTaskStore` 리로드 → React 리렌더로 진행
- 낙관적 업데이트 없음; 모두 `await db.put → reload → setState`

### 6.10 국제화
- 두 로케일(`en`, `ko`)을 가진 i18next
- Sidebar / MenuSheet / SettingsPage / Projects 섹션 문자열은 모두 번역됨
- QuickAddForm, TaskDetailPanel 내부 라벨은 v1에서 의도적으로 미번역 (지연됨)

## 9. 기술적 제약

- **단일 사용자, 단일 디바이스.** 신원 없음, 인증 없음, "현재 사용자"라는 개념 없음.
- **영속성은 IndexedDB만.** 주 저장소로 localStorage 안 씀. sessionStorage 안 씀.
- **새 런타임 의존성 추가 안 함** — 강한 정당화 없으면. 번들은 gzip 600KB 미만 유지.
- **CSS-in-JS 없음.** Tailwind utility 클래스만, `index.css`의 인라인 `@theme` 토큰 블록 제외.
- **글로벌 상태 라이브러리 없음** — React Context 너머. Redux 없음, Zustand 없음, MobX 없음, Jotai 없음.
- **Tailwind v4 인라인 `@theme`** — `tailwind.config.js` 없음. 커스텀 토큰은 `styles/tokens.css`에 살음.
- **단일 브레이크포인트 1024px.** `md:`나 `xl:` 분화 없음. 태블릿은 모바일 UI 렌더.
- **SSR 없음.** Vite SPA 전용. `useMediaQuery` 초기값은 `window.matchMedia` 사용.

## 10. 성공 지표

이 앱은 오프라인 전용이고 텔레메트리가 없기 때문에, 성공은 분석이 아니라 *질적인* 사용자 행동 신호로 정의됩니다.

- **데일리 재사용**: 사용자가 프롬프트 없이도 일주일에 적어도 3일 앱을 연다
- **Today 짧은 리스트 완료율**: 주관적으로 "대부분의 날 내가 고른 걸 끝낸다"
- **분류 부채(Triage debt)**: Inbox에 미분류 task가 항상 20개 미만
- **포모도로 채택**: 활성 일자당 적어도 한 세션
- **Settings 안정성**: 사용자가 포모도로 기본값을 한두 번 바꾸고 그 후로는 절대 안 바꿈

사용자가 다시 종이 노트를 쓰기 시작하면, 제품은 실패한 것입니다.

## 12. 미해결 질문

- 포모도로 타이머가 완료 시 알림/벨을 울려야 할까? Phase 1은 조용히 전환하고, 사용자가 직접 봐야 함. 조용한 실행이 옳은 건지, 아니면 그저 미완성인지 아직 결정되지 않음.
- 완료된 task를 N일 후 Brain Dump에서 페이드아웃 시켜야 할까? 현재는 Completed 섹션에 영원히 머무름.
- "프로젝트 보관 해제(Unarchive)" 경로가 있어야 할까? 현재 보관은 UI에서 일방향; 사용자가 IndexedDB를 직접 편집해야 복원 가능. v1으로는 수용 가능.
